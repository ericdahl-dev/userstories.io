The core wedge is in place:

1. Developer connects a GitHub repo and shares a portal link
2. Collaborator signs in with a magic link and submits a story
3. Developer reviews pending items in the triage inbox
4. Accept creates a GitHub issue with a backlink to userstories.io

This keeps the issue tracker clean while giving non-technical stakeholders a simple front door.
